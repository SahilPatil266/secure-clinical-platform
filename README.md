A Python cryptographic security system built to demonstrate practical application of encryption, access control, and regulatory compliance in a healthcare data context.
The project simulates a real-world engineering problem: how do you securely share sensitive patient data between multiple authorised users across different roles, without shared passwords, without trusting the network, and while satisfying GDPR requirements for special category health data?
The solution implements industry-standard cryptographic primitives — AES-256-GCM, RSA-2048, bcrypt, and SHA-256 — combined with a multi-recipient key encapsulation scheme, a tamper-evident audit log, role-based access control enforced at code level, and a software HSM abstraction layer designed to be swapped out for certified hardware in a production environment.
